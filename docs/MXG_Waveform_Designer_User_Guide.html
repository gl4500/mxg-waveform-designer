1   Quick Start

Windows

Double-click install_and_run.bat in the project folder.

The script will automatically create a Python virtual environment, install all required packages, and launch the GUI. No Anaconda, no pre-installed libraries needed — just Python 3.8 or newer from python.org.

macOS / Linux

bash install_and_run.sh
macOS note — tkinter If the GUI window does not open, install the Tk binding for your Python version:
Homebrew Python: brew install python-tk
Debian / Ubuntu: sudo apt install python3-tk

Manual launch (if packages are already installed)

python mxg_waveform_designer.py

2   What the Tool Does

The MXG Waveform Designer builds composite baseband I/Q waveforms made up of one or more channel banks. Each bank is a group of uniformly-spaced signal channels of the same type. Banks of different types can be combined freely — for example 30 LFM channels plus 8 BPSK channels in a single composite pulse train.

Supported waveform types per channel:

TypeDescription
LFMLinear FM chirp — standard radar pulse compression waveform
NLFMNon-linear FM — lower time sidelobes than LFM without amplitude window loss. Laws: tangent, cosine, hamming
CWContinuous-wave pure tone
FMCWFM-CW sweep — sawtooth or triangle shape
STEPPEDStepped-frequency burst — N equal-dwell CW tones spanning the bandwidth
BPSKBinary phase-coded — Barker-13 by default, or custom chip sequence
FRANKFrank polyphase code — near-ideal thumbtack ambiguity function

The output is saved to a params/ subfolder as binary (.bin), MATLAB (.mat), CSV, and a JSON parameters file. The _mxg_4mb.bin file is sized and formatted for direct loading onto the instrument.

3   Tab 1 — Timing & RF

Timing & RF Sets the timing skeleton and instrument profile for the waveform.

Instrument selector

Choose your instrument from the dropdown at the top of this tab. Selecting an instrument automatically fills the recommended sample rate and displays the RF frequency range, maximum ARB clock, and modulation bandwidth limit in the info row below the dropdown.

Select Custom (manual entry) to enter all values by hand with no automatic limits enforced.

Timing fields

FieldWhat to enter
Sample rate (MHz)ARB sample rate. Auto-filled when an instrument is selected. Must not exceed the instrument's ARB clock maximum or the bandwidth check will block the build.
Pulse width (µs)Duration of the active signal within each PRI. Must be strictly less than the PRI.
PRI (µs)Pulse Repetition Interval — total length of one pulse slot including dead time after the pulse.
Num pulsesNumber of PRIs concatenated to form the full output waveform. More pulses = larger file.
RF centre (GHz)Notional carrier frequency. Used for metadata and the MATLAB .m script only — does not affect baseband generation.
Rule of thumb for pulse width and PRI A duty cycle of 50 % (pulse width = half the PRI) is a good starting point. E.g. pulse width 1000 µs, PRI 2000 µs.

4   Tab 2 — Channels

Channels Defines which signals make up the composite waveform.

Bank list

The top section shows all currently defined channel banks in a table. Each row is one uniform group of channels. At launch a default 40-channel LFM bank is pre-loaded.

ButtonAction
Add bankReads the entry form below and appends a new bank. You can add as many banks as needed — they are all combined into the final waveform.
Remove selectedClick a row in the list to highlight it, then click this to delete that bank.

New bank entry form

FieldDescription
Waveform typeSelect one type for this bank. Each bank can be a different type.
Num channelsHow many individual channels in this bank. They are placed at uniform frequency spacings centred on DC.
Chan spacing (MHz)Frequency separation between adjacent channel centres. Make sure spacing ≥ bandwidth to avoid channel overlap.
Bandwidth/ch (MHz)Chirp or sweep bandwidth per channel. Not used for CW channels.
Up-chirpTick for an up-sweep, untick for a down-sweep. Applies to LFM, NLFM, FMCW.
Waveform optionsExtra fields that appear depending on the type selected:
NLFM: Law (tangent / cosine / hamming), Beta shaping factor
FMCW: Shape (sawtooth / triangle)
STEPPED: Number of frequency steps
BPSK: Chip rate (MHz)
FRANK: Order M (M² total chips), Chip rate (MHz)

Bandwidth check bar

Updates live as you change channel parameters. Shows the occupied span of the current bank entry vs the selected instrument's modulation bandwidth limit.

  • Green ✓ — within the instrument's modulation bandwidth limit.
  • Red ⚠ — exceeds the limit. The build will be blocked until you reduce channels, spacing, or bandwidth.

The "Max channels" hint shows how many channels will fit at the current spacing before hitting the limit.

Occupied bandwidth formula Occupied span = (N − 1) × spacing + per-channel bandwidth.
For 40 channels at 3 MHz spacing with 2 MHz bandwidth: (39 × 3) + 2 = 119 MHz. This exceeds the N5182A's 100 MHz limit — reduce to 32 channels at 3 MHz spacing to fit.

5   Tab 3 — Window & Scale

Window & Scale Controls envelope shaping and channel amplitude weighting.

Pulse envelope window

FieldDescription
Apply pulse windowWhen ticked, multiplies the pulse time-domain envelope by the chosen window function to smooth the on/off transitions and reduce spectral sidelobes.
Window typehann (default) — good all-round sidelobe suppression.
hamming — slightly higher sidelobes, slightly better main-lobe width.
tukey — rectangular in the middle, cosine tapered at the edges. Tukey alpha controls the taper fraction (0 = rectangular, 1 = Hann).
none — no windowing applied.
Tukey alphaOnly active when window type is tukey. Range 0–1. Default 0.25 gives a 25 % taper.

Peak scale

Sets the final peak amplitude of the composite waveform as a fraction of full scale (0–1). Default is 0.85, leaving about 1.4 dB of headroom below the instrument's ±32767 clipping point. Reduce this value if you see scale utilisation warnings in the diagnostic plots.

Per-channel phase dither

When Random per-channel phase is ticked, each channel is assigned a random initial phase offset. This breaks the constructive interference that would otherwise cause a very high crest factor (PAPR) in a many-channel composite. The RNG seed value makes the random draw repeatable — the same seed always produces the same phase set.

Channel amplitude taper

Applies a cross-channel amplitude weighting across the bank so that edge channels are attenuated relative to centre channels. Useful for reducing spectral spillover outside the composite bandwidth.

FieldDescription
Channel amplitude taperTick to enable. Applies the taper defined below across all channels in order of frequency.
Taper typecosine — simple cosine-squared roll-off between edge and centre scales.
taylor — true Taylor window with specified sidelobe level and nbar. Better sidelobe control than cosine.
chebwin — Chebyshev window with equiripple sidelobes at exactly the specified level.
Taylor nbarNumber of constant-level inner sidelobes adjacent to the main lobe. Typical value 4–6. Higher = sharper main lobe. Used by taylor and chebwin.
SLL (dB, neg.)Desired sidelobe level. Enter a negative number, e.g. -30 for −30 dB suppression. Used by taylor and chebwin.

7   Tab 5 — SCPI Download

SCPI Transfers a _mxg_4mb.bin file directly to a connected N5182A over LAN or GPIB without manually copying files.

Requires PyVISA Install it once before using this tab:
pip install pyvisa pyvisa-py
The tab shows an install reminder if pyvisa is not found.
FieldWhat to enter
VISA resourceThe instrument's address string. LAN example: TCPIP0::192.168.1.100::5025::SOCKET. GPIB example: GPIB0::19::INSTR. Find the address in the instrument's I/O config menu or using Keysight Connection Expert.
Waveform nameName the waveform will have in the instrument's catalog, e.g. my_lfm_composite. No spaces or special characters.
Sample rate (MHz)Sample rate to set on the ARB after transfer. Must match the value in the _mxg_4mb_info.txt sidecar file.
Bin file pathFull path to the _mxg_4mb.bin file. Click Browse… to navigate to it.

Click Connect & Download to N5182A. The status bar below the button shows connection progress, then transfer progress, then either a green "Done" confirmation or a red error message.

What the download does automatically:

  1. Transfers the .bin file using the SCPI MMEM:DATA block-data command.
  2. Selects the waveform: WGEN:ARB:WAVEFORM "WFM1:<name>"
  3. Sets the sample rate: WGEN:ARB:SRAT <rate>
  4. Enables ARB output: :OUTPUT:STATE ON
Manual SCPI option A _mxg_4mb_scpi.txt file is always saved alongside the .bin. It contains the exact SCPI command sequence in both Python/PyVISA form and raw form for use with Keysight IO Libraries Suite or any other VISA client.

8   Step-by-Step Build Workflow

  1. Select your instrument on the Timing & RF tab. The sample rate auto-fills. Set your pulse width, PRI, and number of pulses.
  2. Define channel banks on the Channels tab. Adjust the default LFM bank or remove it and add new banks. Watch the bandwidth check bar — it must be green before you can build.
  3. Set windowing and taper on the Window & Scale tab. For most cases the defaults (Hann window, no taper, peak scale 0.85) work well.
  4. Configure output on the Output tab. Click Browse… to pick a save folder, then click Auto-name to generate a unique filename.
  5. Click Build Waveform. The button shows "Building…" and the status bar at the bottom of the window shows progress. The GUI stays open and responsive.
  6. Review the plots if "Show diagnostic plots" is ticked. The time-domain, spectrum, and spectrogram windows open and can be moved, zoomed, and saved.
  7. Load onto the instrument — either use the SCPI tab, or copy the _mxg_4mb.bin file manually via FTP/USB and follow the instructions in the _mxg_4mb_scpi.txt file.
  8. Build again without restarting — click Auto-name again for a new timestamp, adjust any parameters, and click Build. Previous files are never overwritten.

9   Output Files Explained

All files are written to <output folder>/params/.

params/
├── LFM_40ch_125MHz_1000us_20pri_20260405_143211_full.bin ← full-res archive
├── LFM_40ch_125MHz_1000us_20pri_20260405_143211_mxg_4mb.bin ← load this onto the MXG
├── LFM_40ch_125MHz_1000us_20pri_20260405_143211_mxg_4mb_info.txt ← sample rate to set
├── LFM_40ch_125MHz_1000us_20pri_20260405_143211_mxg_4mb_scpi.txt ← SCPI command sequence
├── LFM_40ch_125MHz_1000us_20pri_20260405_143211.mat ← MATLAB archive
├── LFM_40ch_125MHz_1000us_20pri_20260405_143211.csv ← raw I/Q text
├── LFM_40ch_125MHz_1000us_20pri_20260405_143211_channel_table.csv ← per-channel metadata
└── LFM_40ch_125MHz_1000us_20pri_20260405_143211_params.json ← always saved

Binary file format (N5182A)

PropertyValue
Data typeint16 (signed 16-bit integer)
Byte orderBig-endian (MSB first)
InterleaveI₀, Q₀, I₁, Q₁, … (sample-interleaved)
Full scale±32767
File extension.bin

_params.json contents

The JSON file records everything needed to reproduce the build exactly:

  • Full CompositeConfig — every timing and scaling parameter
  • Full channel list — type, centre frequency, bandwidth, and all type-specific parameters for every channel
  • Derived statistics — total samples, waveform duration, PAPR, channel type counts
  • Build timestamp and output directory

10   Validating a .bin File

Run mxg_bin_validate.py from the command line to check any .bin file before loading it onto the instrument.

python mxg_bin_validate.py params/<stem>_mxg_4mb.bin --fs 125

The tool prints a report showing:

  • File size and IQ pair count
  • Endianness check — confirms the file is big-endian as required by the N5182A
  • I and Q min/max values and peak amplitude
  • Scale utilisation — what percentage of ±32767 full scale is used
  • PAPR (Peak-to-Average Power Ratio) in dB
  • Any warnings (wrong endianness, low scale utilisation, file too large)

It also opens a four-panel diagnostic plot:

  • Time domain — I and Q waveforms with full-scale markers
  • IQ constellation — scatter plot of I vs Q
  • Amplitude envelope — magnitude |I + jQ|
  • Welch PSD — averaged power spectral density in dB/Hz

Command-line options:

FlagDescription
--fs 125Sample rate in MSa/s. Default 125. Must match the value in _mxg_4mb_info.txt.
--info <file>Path to the _info.txt sidecar. Auto-detected if omitted.
--no-plotPrint the report only, skip plots.

11   Reproducing a Build on Another Machine

Copy the params/ folder (or just the _params.json file) to the other machine, then run:

from mxg_waveform_designer import load_params, CompositeBuilder, WaveformExporter

cfg, channels = load_params('params/LFM_40ch_125MHz_1000us_20pri_<ts>_params.json')
iq, table = CompositeBuilder(cfg, channels).build()
WaveformExporter().save_all(iq, cfg, channels, table)

This reconstructs the identical waveform from the saved parameters without needing to open the GUI.
